# Прокси для Gemini Live API

Этим воркером и сервисом пользуются те, у кого Google AI API заблокирован
напрямую (Беларусь, Россия, Иран, Куба и пр.). В Liv в настройках есть
переключатель «Маршрут (для стран с блокировкой)», в который вписывается
адрес такого прокси — и Liv ходит к Google не напрямую, а через него.

В этой папке лежит **два кусочка**:

- [`cloudflare-worker.js`](./cloudflare-worker.js) — публичный фронт прокси
  на домене `*.workers.dev`. Этот домен не блокируется белорусскими и
  российскими провайдерами, поэтому браузер пользователя до него доходит.
- [`fly-app/`](./fly-app) — Python-сервис на Fly.io, который реально ходит
  к Google. Его поднимаем в выбранном регионе (US/EU), чтобы Google видел
  «разрешённый» IP.

Для большинства провайдеров с DPI-блокировкой нужны **оба**: воркер
проксирует браузер на Fly.io-приложение (потому что сами `*.fly.dev`
домены тоже часто заблокированы), а Fly.io-приложение — уже в Google.

## Деплой

### 1. Fly.io приложение

1. Установи [`flyctl`](https://fly.io/docs/flyctl/install/):
   `curl -L https://fly.io/install.sh | sh`.
2. `fly auth signup` (или `fly auth login`, если уже есть аккаунт).
3. Из папки `proxy/fly-app/`:
   ```bash
   fly launch --no-deploy
   # выбери имя приложения (например `livvv-proxy-<суффикс>`),
   # выбери регион (sjc/iad/fra/lhr — главное, чтобы Google его пропускал),
   # откажись от Postgres и Redis.
   fly deploy
   ```
4. Открой `https://<имя-приложения>.fly.dev/health` — должно вернуться
   `Liv proxy is alive...`.
5. Открой `https://<имя-приложения>.fly.dev/debug` — должно показать
   `loc=US` (или другую разрешённую страну) в исходящем трейсе.

> **Важно**: домен `*.fly.dev` часто блокируется на уровне провайдера.
> Если /debug у тебя из заблокированного региона не открывается — это
> ожидаемо. Идём дальше: воркер на Cloudflare умеет проксировать в
> fly.io даже когда твой провайдер не пускает.

### 2. Cloudflare Worker

1. Зарегистрируйся на [dash.cloudflare.com](https://dash.cloudflare.com/sign-up)
   (бесплатно, без карты). Если дашборд CF у тебя из заблокированного
   региона не открывается, включи на минуту VPN, чтобы выполнить шаги 2–4.
2. **Workers & Pages → Create Worker → Start with Hello World!** Назови
   воркер (например `livvv-proxy`) → **Deploy**.
3. Открой воркер → **`</> Edit code`** → удали дефолтный код, вставь
   содержимое [`cloudflare-worker.js`](./cloudflare-worker.js).
4. Поправь сверху файла строку `const UPSTREAM_HOST = '...'`. Подставь
   адрес своего Fly.io-приложения (например
   `livvv-proxy-abc123.fly.dev`). Жми **Deploy**.
5. Cloudflare покажет URL воркера, например
   `https://livvv-proxy.<твой-subdomain>.workers.dev`.
6. Проверь: открой
   `https://livvv-proxy.<твой-subdomain>.workers.dev/debug` — должно
   показать секции `inbound`, `outbound` и `upstream (...) /health` со
   статусом `200 Liv proxy is alive...`. Это значит, что воркер успешно
   достукивается до Fly.io.

### 3. Использование в Liv

В настройках Liv → раздел «Маршрут (для стран с блокировкой)»:

- Включи переключатель «Подключаться через прокси».
- В поле «Адрес прокси» вставь адрес **Cloudflare Worker** (не Fly.io):
  `livvv-proxy.<твой-subdomain>.workers.dev` (без `https://`, но если
  вставишь с протоколом — он автоматически уберётся).

Перезапусти сессию. В журнале событий должна появиться строка
«Соединение через прокси: ...» и тут же «Подключение к Gemini Live
установлено».

## Если хочется проще (один Cloudflare Worker, без Fly.io)

Если у твоего провайдера `*.fly.dev` доступен (или ты живёшь не за DPI),
можно обойтись одним воркером. В таком случае в `cloudflare-worker.js`
поставь:

```js
const UPSTREAM_HOST = 'generativelanguage.googleapis.com';
```

И включи в настройках воркера на dashboard.cloudflare.com:
**Settings → Placement → Smart**. Cloudflare попробует запускать твой
воркер в дата-центре поближе к Google. Это срабатывает не всегда —
если /debug показывает `loc=BY`/`loc=RU`/`loc=PL` (страны, которые
Google блокирует), переключайся на режим с Fly.io.

## Лимиты

- Cloudflare Workers Free: **100 000 запросов в день**. Каждая
  WebSocket-сессия — это 1 запрос с точки зрения CF, аудиочанки внутри
  не считаются. Для личного использования — потолка не достичь.
- Fly.io Free: 3 машины × 256MB RAM. Этого хватает на одного-двух
  пользователей. Машины уходят в auto-suspend, когда никто не подключён,
  и просыпаются при первом запросе (~1 секунда холодного старта).

## Безопасность

И воркер, и Fly.io-приложение — это публичные URL без аутентификации.
Если кто-то узнает адрес твоего прокси, он сможет ходить через него к
Gemini API со своим собственным API-ключом. Это не утечка твоего ключа
(ключ остаётся в браузере у каждого пользователя), но это «бесплатный
туннель», который ты раздаёшь.

Если хочется ограничить — добавь в воркер проверку секретного заголовка
или конкретного origin (например `*.vercel.app`). Для текущей задачи
(личный проект, который иногда показываешь друзьям) — это нестрашно.
